A especificação HTML5 traz diversos novos elementos aos desenvolvedores web, permitindo-os descrever a estrutura de um documento web com semânticas padronizadas.

Estrutura de um documento em HTML4

A estrutura de um documento, ou seja, a estrutura semântica do que está entre <body> e </body>, é fundamental para a apresentação da página ao usuário. O HTML4 usa a noção de seções e subseções de um documento para descrever sua estrutura. Uma seção é definida por um Elemento HTML de Divisão (<div>) com Elementos HTML de Cabeçalho (<h1>, <h2>, <h3>, <h4>, <h5>, ou <h6>) dentro de si, definindo seu título. As relações desses elementos levam à estrutura do documento.

Elementos Estruturais

Novos elementos no HTML5

Os seguintes elementos foram introduzidos para melhor estrutura:

  • section representa uma seção genérica de documento ou aplicativo. Deve ser usado junto com os elementos h1, h2, h3, h4, h5 e h6 para indicar a estrutura do documento.
  • article representa uma parte independente do conteúdo de um documento, como uma entrada de blog ou artigo de jornal.
  • main representa o conteúdo principal do corpo de um documento ou aplicativo.
  • aside representa um conteúdo que está apenas ligeiramente relacionado ao restante da página.
  • header representa um grupo de auxílios introdutórios ou de navegação.
  • footer representa um rodapé de uma seção e pode conter informações sobre o autor, informações sobre direitos autorais etc.
  • nav representa uma seção do documento destinado à navegação.
  • figure representa uma parte do conteúdo do fluxo independente, geralmente referenciado como uma única unidade do fluxo principal do documento.
    •             <figure>
                <video src="example.webm" controls</video>
                <figcaption<Example</figcaption>
            </figure>
  • figcaption pode ser usado como legenda (é opcional).
  • template pode ser usado para declarar fragmentos de HTML que podem ser clonados e inseridos no documento por script.

Há também muitos outros elementos:

  • video e audio para conteúdo multimídia. Ambos fornecem uma API para que os desenvolvedores da Web de aplicativos possam criar scripts para sua própria interface de usuário. a tag source é usada com estes elementos se existem diferentes tipos de streaming disponíveis.
  • track providencia elementos de texto no vídeo como captions e legendas.
  • embed é usada para conteúdo de plug-ins.
  • mark representa uma sequência de texto em um documento marcado ou destacado para fins de referência, devido à sua relevância em outro contexto.
  • task representa a conclusão de uma tarefa, como o download ou a execução de uma série de operações
  • meter representa uma medida, como o uso do disco.
  • time representa uma data e/ou hora.
  • ruby, rt e rp permitem marcar anotações em ruby.
  • bdi representa um espaço de texto que deve ser isolado do ambiente ao redor para fins de formatação de texto bidirecional.
  • wbr representa uma oportunidade de quebra de linha.
  • canvas é usada para renderizar gráficos dinâmicos de bitmap em tempo real, como gráficos ou jogos.
  • datalist juntamente com o novo atributo list para input, pode ser usado para criar caixas de combinação:
    •             <input list="browsers">
            <datalist id="browsers">
                <option value="Safari">
                <option value="Internet Explorer">
                <option value="Opera">
                <option value="Firefox">
            </datalist>
  • keygen representa o controle para a geração de pares de chaves.
  • output representa algum tipo de output, como em um cálculo feito por meio de script.
  • Os atributos do tipo input agora possuem os seguintes valores:
    • tel
    • search
    • url
    • email
    • date
    • time
    • number
    • range
    • color

A idéia desses novos tipos é que o agente do usuário possa fornecer a interface do usuário, como um selecionador de data do calendário ou a integração com o catálogo de endereços do usuário, e enviar um formato definido ao servidor. Isso proporciona ao usuário uma experiência melhor, pois sua entrada é verificada antes de enviá-la ao servidor, o que significa que há menos tempo para aguardar feedback.

Novos atributos

Vários atributos foram introduzidos em vários elementos que já faziam parte do HTML4:

  • O elemento area, para consistência com os elementos a e link, agora também possui os atributos hreflang, type e rel.
  • O elemento base agora também pode ter um atributo target, principalmente para consistência com o elemento a. (Isso já é amplamente suportado.)
  • O elemento meta agora possui um atributo charset, já que isso já é amplamente suportado e fornece uma boa maneira de especificar a codificação de caracteres para o documento.
  • Um novo atributo autofocus pode ser especificado nos elementos input (exceto quando o atributo type é hidden), select, textarea e button. Ele fornece uma maneira declarativa de focar um controle de formulário durante o carregamento da página. O uso desse recurso deve aprimorar a experiência do usuário, em comparação com o foco no elemento com o script, pois o usuário pode desativá-lo se não gostar, por exemplo.
  • Um novo atributo placeholder pode ser especificado nos elementos input e textarea. Representa uma dica destinada a auxiliar o usuário na entrada de dados.
    •             <input type=search name=q placeholder="Enter search phrase..."> <button<Search</button>
            <label>Email <input type=email name=email placeholder="john@example.com"></label>
